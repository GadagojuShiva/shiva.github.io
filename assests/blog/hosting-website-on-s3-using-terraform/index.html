Hosting a Static Website on AWS S3 using Terraform

In this comprehensive guide, we'll take you through the process of hosting a static website on Amazon S3 using Terraform. Follow these step-by-step instructions, assuming you have Terraform installed on your Windows machine and an active AWS account.

Prerequisites:

Before diving into the setup, ensure you have the following prerequisites:

  • Terraform installed on Windows: Install Terraform
  • AWS account: Create AWS Account

Step 1: Create an S3 Bucket

The first step involves creating an S3 bucket using Terraform. S3 buckets are versatile storage containers in AWS, and they will serve as the foundation for hosting your static website files. In the main.tf file, you define an AWS S3 bucket using Terraform's aws_s3_bucket resource block. This sets up the core storage for your website.

Step 2: Set Ownership Controls:

In this step, you extend the main.tf file to include ownership controls for the S3 bucket. Ownership controls ensure that objects within the bucket are associated with the preferred owner. This is achieved by using the aws_s3_bucket_ownership_controls resource block.

Step 3: Make the Bucket Public:

Continuing in the main.tf file, this step focuses on configuring public access to the S3 bucket. By using the aws_s3_bucket_public_access_block resource block, you control various aspects of public access, such as blocking public ACLs and policies.

Step 4: Configure Bucket ACL:

This step involves setting up Access Control Lists (ACL) for the S3 bucket. The aws_s3_bucket_acl resource block in the main.tf file allows you to define the ACL for the bucket, ensuring it is set to "public-read."

Step 5: Upload Website Files

Extending the main.tf file, you configure Terraform to upload your website files to the S3 bucket. Using aws_s3_objectresource blocks, you specify the key, source path, ACL, and content type for each file (e.g., index.html, error.html).

Step 6: Configure Website Hosting:

The final step in the main.tf file involves configuring website hosting for the S3 bucket. By using the aws_s3_bucket_website_configuration resource block, you define the index document (e.g., index.html) and error document (e.g., error.html), making your S3 bucket suitable for hosting a static website.
